# 产品需求文档 (PRD): 智能科研论文分析工具2.0

## 1. 产品概述
**产品名称**：PaperInsight CLI (暂定)
**产品定位**：一款跨平台的命令行工具，专为光电/半导体领域的科研人员设计。通过集成百度 PaddleOCR API、大模型（LLM）能力以及本地正则解析脚本，自动化批量阅读 PDF 论文，精准提取关键实验参数（EQE、CIE等）及策略描述，并自动关联影响因子（IF），最终生成高质量、按影响因子排序的 Excel 调研报告。

## 2. 核心功能与交互流程

### 2.1 启动与配置引导 (CLI Design)
1. **环境自检**：启动时检测配置文件（如 `config.yaml`）。若缺少必要的 API Key（Baidu OCR API / LLM API），则通过 CLI 提示用户输入并持久化保存。
2. **模式选择**：
   * **智能 API 模式**：调用百度 OCR + LLM 进行高精度语义提取（消耗 API 额度）。
   * **基础正则模式**：调用已有的本地 Python 正则脚本进行极速提取（兜底方案，免费）。
3. **任务输入**：支持用户直接拖拽包含 PDF 的文件夹到终端，或手动输入绝对/相对路径。

### 2.2 核心执行管线 (Pipeline)
1. **文件扫描与排重（本地缓存层）**：
   * 对扫描到的每一个 PDF 计算 **MD5 哈希值**作为唯一指纹。
   * 检查本地隐藏缓存目录（如 `.cache/`）：
     * 若存在 `[MD5]_data.json`，直接读取已提取的结构化数据，跳过分析。
     * 若存在 `[MD5]_ocr.txt`，跳过 OCR 阶段，直接将其送入 LLM 进行分析。
2. **智能文本提取（省流优化）**：
   * 优先尝试使用基础库（如 PyMuPDF）读取原生 PDF 文本。
   * 若文本提取失败或乱码（说明是图片扫描件），则调用百度 OCR API 进行全文/表格识别。由于使用的 OCR 服务原生支持全文处理，直接获取完整版面文本并存入本地 `[MD5]_ocr.txt`，方便用户日后直接复制。
3. **大模型语义抽取 (LLM Processing)**：
   * 将获取到的全文/表格文本送入 LLM。
   * 提取指定字段，要求 LLM 输出标准化的 JSON 格式。
4. **影响因子 (IF) 补全**：
   * 从提取结果中获取“期刊名称”。
   * 若 PDF 内未包含影响因子，程序自动调用 Web 搜索工具/API，在网上查找该期刊的最新 IF 值并补充。

### 2.3 异常隔离与断点续传
* **容错机制**：批量处理中，若单篇论文因网络超时、API 额度耗尽或文件损坏报错，程序捕获异常，在终端输出红色警告（如 `[Skipped] paper_A.pdf: API Quota Exceeded`），**跳过该文件并继续执行剩余任务**。
* **错误日志**：运行结束后，在目标文件夹生成 `error_log.txt` 记录失败详情。

## 3. 数据提取与输出规格

### 3.1 字段提取要求
| 提取字段 | 提取逻辑与要求 | 备注 |
| :--- | :--- | :--- |
| **期刊名称** | 从首页或页眉页脚提取 | 用于 IF 匹配 |
| **影响因子 (IF)** | 优先匹配 PDF 原文 -> 其次通过 Web 联网查询补全 | 核心排序依据 |
| **论文标题 & 作者** | 识别第一页最大字号文本及署名 | - |
| **器件结构** | 识别层级堆叠（如 `ITO/HTL/EML...`） | LLM 进行格式规整 |
| **实验参数 (EQE/CIE/寿命)**| 识别对应数值。如果一篇论文包含多个器件数据，需**并列存放在同一个单元格内**（以换行符分隔） | - |
| **数据溯源 (新增)** | 针对提取的数值，附带原文出处句子，如 `...max EQE of 20.5%...` | 提高数据可信度 |
| **优化层级和策略** | 利用 LLM 结合全文，总结提取论文中的“优化方法与策略” | 字数限制在 100 字左右 |

### 3.2 报表与文件输出
* **Excel 报表生成**：
  * 在输入的文件夹下生成 `Paper_Analysis_Report.xlsx`。
  * **必须按照“影响因子 (IF)”从高到低对所有行进行降序排列**。
  * 自动调整列宽，设置文本自动换行，确保多器件参数显示美观。
* **文件自动重命名归档 (Bonus)**：
  * 提供可选项，将处理完的 PDF 文件重命名为规范格式，如：`[年份_IF影响因子_期刊名]_论文标题.pdf`。

## 4. 技术栈与兼容性要求
* **开发语言**：Python 3.8+
* **跨平台兼容**：
  * 使用 `pathlib` 处理 Windows (`\`) 与 macOS/Linux (`/`) 的路径分隔符差异。
  * 文件读写强制使用 `utf-8` 编码，避免 Windows 下的 GBK 报错。
* **依赖库推荐**：
  * CLI 框架：`Click` 或 `Typer`
  * 进度展示：`tqdm` (提供美观的 CLI 进度条)
  * PDF 处理：`PyMuPDF` (基础解析) / `pdf2image` (转图)
  * 数据处理与导出：`pandas`, `openpyxl`
