# 产品需求文档 (PRD): PaperInsight v3.0 (科研论文智能解析引擎)

## 1. 产品概述
**PaperInsight v3.0** 是一款面向科研人员的自动化论文数据解析工具。通过将 PDF 论文解析为高质量 Markdown，结合大语言模型 (LLM) 进行深度语义推理，从复杂的科研论文中精准提取并结构化实验参数（器件结构、EQE、CIE、寿命等），生成科研调研 Excel 报告。

## 2. 核心架构升级 (The "3.0" Pipeline)
1. **文档标准化层 (MinerU)**：将 PDF 高精度转化为 Markdown，保留表格层级与排版特征，剔除干扰信息。
2. **文本降噪层 (Cleaning)**：自动过滤参考文献、鸣谢、作者贡献等高噪声章节，重点提取 `Abstract`, `Experimental Section`, `Results and Discussion`。
3. **LLM 语义推理层 (Extraction)**：使用嵌套式 JSON Schema 强制规范大模型输出，确保实验参数与器件结构的强绑定，解决“一文多数据”提取错位问题。
4. **校验与导出层 (Validation & Export)**：通过 Pydantic 校验模型返回结构，并按“每篇论文一行，单元格内换行拼接”的格式输出。

## 3. 详细功能需求

### 3.1 智能化引擎逻辑
* **PDF 解析**：优先使用 `MinerU` 将 PDF 转为 Markdown，直接获取表格数据结构。
* **数据嵌套逻辑**：LLM 输出必须遵循以下 JSON 数据结构，以确保数据完整性：
    * `paper_info`: 包含 `optimization_strategy` (策略总结) 及 `best_eqe` (最高值)。
    * `devices` (List): 每个列表项包含 `device_label`, `structure`, `EQE`, `CIE`, `lifetime`, `notes`。
* **数据溯源**：LLM 提取每个参数时，需同时返回一段简短的原文引用，提高数据可信度。

### 3.2 交互与运行策略
* **缓存优先 (Cache-first)**：基于文件 MD5 指纹进行缓存，确保相同论文无需二次调用 MinerU 和 LLM，显著节省 API 成本。
* **异常处理**：采用异步/分片执行，单个文件解析失败不中断整体进程，在 `error_log.txt` 中详细记录错误原因。
* **IF 自动补全**：若 PDF 无 IF 信息，通过期刊名触发 Web 查询/爬虫补充。

### 3.3 结果呈现 (Excel 规格)
* **排序**：Excel 报表严格按 `影响因子 (IF)` 降序排列。
* **合并模式**：
    * **主信息列**（标题、作者、IF）独占单元格。
    * **多器件数据列**（结构、EQE、CIE）：若存在多个器件，使用 `\n` 换行符在同一个单元格内展示多个记录（例如：`[Control] 18.2% \n [Champion] 22.5%`）。

## 4. 关键技术指标
| 模块 | 技术实现 | 目的 |
| :--- | :--- | :--- |
| **文档转换** | `MinerU` / `PDF-Extract-Kit` | 将表格还原为 Markdown 表格，非纯文本流 |
| **结构化推理** | `LLM (文心/GPT/DeepSeek)` | 自动理解上下文，完成 JSON 映射 |
| **数据校验** | `Pydantic` (Python 库) | 确保 API 返回的数据格式符合预期，防止解析崩溃 |
| **持久化** | `MD5 Hash + SQLite/JSON` | 实现缓存机制，节省费用 |

### 给开发者的执行建议 (v3.0 Action Plan)：

1. **核心重构**：将现有的 `OCR 提取逻辑` 替换为 `MinerU 命令行调用逻辑`。MinerU 安装后直接输出 `.md` 文件，你只需读取该文件。
2. **Prompt 工程**：你的 Prompt 必须包含：“请将此 Markdown 论文分析为以下 JSON 结构：`{ "paper_info": ..., "devices": [ { "structure": ..., "EQE": ... } ] }`”。
3. **数据校验层**：在 Python 端引入 `from pydantic import BaseModel`，定义好你的 `DeviceData` 类和 `PaperData` 类。这会极大提升工具的稳定性。
