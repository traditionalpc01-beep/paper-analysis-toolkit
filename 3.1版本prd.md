# PaperInsight v3.1 产品需求文档（结构感知过滤与结果矫正版）

## 1. 版本目标

PaperInsight v3.1 是在 v3.0（MinerU + LLM 嵌套提取）基础上的一次工程化升级，重点解决以下四类问题：

1. 章节白名单过滤过粗，容易遗漏散落在摘要、图注、表格上下文中的关键参数。
2. 清洗后的文本仍可能过长，导致 LLM 出现上下文注意力衰减，甚至因为硬截断而丢数据。
3. 影响因子列存在错误或缺失，现有网页解析策略不稳定。
4. 双语输出体验不足：
   - 每次运行前不能明确确认是否需要双语；
   - 报表中的中英对照仅通过 `|` 分隔，可读性弱；
   - 中文部分没有视觉强调。

v3.1 的目标不是继续做“章节级保留”，而是升级为“Markdown 结构感知 + 块级打分 + 邻域窗口保留 + 严格结构化输出”的完整链路。

---

## 2. v3.1 总体目标

### 2.1 精度目标
- 保留论文中与器件结构、EQE、CIE、寿命、效率、亮度等相关的高价值上下文。
- 优先保留表格、图注及其邻接说明文本，减少参数孤立现象。
- 降低多器件论文中的字段串位风险。

### 2.2 成本目标
- 在尽可能不丢失有效参数的前提下，将送入 LLM 的文本压缩到预算范围内。
- 避免在 `extractor` 阶段进行粗暴固定长度截断。
- 让 Prompt 模板结构稳定，提升支持平台的 Prompt Cache 命中机会。

### 2.3 可用性目标
- 在每次处理前都由用户确认是否需要中英双语对照输出。
- 中英双语输出采用分段形式，而不是 `中文 | English` 的单行拼接。
- 在 Excel 报表中，对双语字段的中文部分进行加粗显示。

### 2.4 数据修正目标
- 当论文中影响因子缺失或明显异常时，启用联网校正机制补全或纠正。
- 联网校正结果应优先用于修复缺失值，并在与原值偏差明显时矫正错误值。

---

## 3. v3.1 核心处理流程

### 阶段一：Markdown 结构感知预处理

输入为 MinerU 输出的完整 Markdown。

处理规则：
1. 不再按章节白名单保留全文。
2. 仅从文末向前识别并截断绝对噪声章节：
   - References / Bibliography
   - Acknowledgements
   - Author Contributions
   - Data Availability
   - Conflict of Interest
   - Appendix
3. 将 Markdown 拆分为结构化 block：
   - heading
   - paragraph
   - list
   - table
   - figure_caption
4. 表格进行锚点化处理：
   - 将原始表格保留为独立对象；
   - 在正文块中替换为 `[TABLE_0001]` 这类稳定占位符；
   - 同时记录表格所属标题路径、表前上下文、表后上下文。

### 阶段二：块级打分与窗口扩展

对每个 block 进行综合打分，而不是简单做“是否命中关键词”的二值判断。

评分依据：
1. 参数关键词命中：EQE、CIE、lifetime、T50、T90、luminance、current efficiency 等。
2. 结构关键词命中：device、structure、ITO、PEDOT:PSS、HTL、ETL、EML、host、dopant 等。
3. 数值/单位模式命中：
   - `%`
   - `cd/A`
   - `lm/W`
   - `cd/m2` / `cd/m²`
   - `mA/cm2` / `mA/cm²`
   - `V`
   - `(0.xx, 0.xx)`
   - `LT50/T50/T90 = xx h`
4. 标题路径加权：
   - Results / Discussion / Experimental / Device Performance 等高价值标题下的 block 加权提高。
   - Introduction / Background 等背景章节降低权重。
5. 图注和表格上下文加权：
   - `figure_caption` 和表格锚点拥有更高优先级。

保留策略：
- 当某个 block 达到最小保留分数后，不仅保留该 block，还保留其前后邻近 block。
- 若命中块位于表格附近，则表格锚点、前后说明文本一并保留。
- 若命中块隶属于某个标题路径，则相关 heading 也应一并保留，保证上下文结构完整。

### 阶段三：预算内重组

将保留的 block 重组成送入 LLM 的 `Condensed Markdown`。

重组规则：
1. 优先输出 Anchored Tables 区域，包含：
   - 表格锚点 ID
   - 标题路径
   - 表前上下文
   - 表格正文
   - 表后上下文
2. 然后输出 Condensed Context 区域，保留命中的核心正文块。
3. 若总长度超出预算：
   - 优先保留 table / figure_caption / 高分参数块；
   - 低分背景块优先剔除；
   - 不再在 extractor 阶段直接硬截断固定 15000 字符。

### 阶段四：严格结构化提取

1. LLM 继续输出 `paper_info + devices + data_source + optimization` 的嵌套 JSON。
2. OpenAI Provider 优先使用严格 `json_schema` 输出模式。
3. 其他 Provider 保持兼容：如不支持严格 schema，则回退到 JSON Object 或文本中抽取 JSON。

### 阶段五：影响因子联网校正

1. 当期刊名称存在时，启用联网查找。
2. 若现有影响因子缺失，则直接补全。
3. 若现有影响因子明显异常（如 <= 0、超范围，或与联网结果差异显著），则用联网结果矫正。
4. 联网策略采用：
   - 基于期刊名搜索候选页面；
   - 匹配最相近的期刊详情页；
   - 从历史影响因子区域中提取最近可用值。

说明：由于公开免费的官方 JCR API 并不稳定可用，v3.1 将该能力定义为 best-effort 校正，而非官方权威源直连。

### 阶段六：双语输出确认与报表格式增强

1. 每次运行前都要询问用户是否启用双语输出，默认按中文处理。
2. 若启用双语输出：
   - 不再使用 `中文 | English` 单行格式；
   - 改为两行：
     - `中文：...`
     - `English: ...`
3. Excel 导出时：
   - 中文行加粗；
   - 英文行保留常规字体；
   - 单元格继续启用换行显示。

## 4. 关键模块改造方案

### 4.1 `paperinsight/cleaner/section_filter.py`

#### 从 v3.0 的章节保留器升级为 v3.1 的块级过滤器

新增能力：
- Markdown block 切分
- 表格锚点化
- 块级评分
- 邻域窗口扩展
- 预算内重组

新增配置项：
```yaml
cleaner:
  enabled: true
  block_window: 1
  max_input_chars: 24000
  max_blocks: 80
  min_block_score: 3.0
  keep_table_context: true
```

### 4.2 `paperinsight/core/extractor.py`

改造目标：
- 不再默认硬截断固定长度字符；
- 优先相信 cleaner 的预算控制；
- 仅在极端情况下做边界式兜底截断；
- 调用 LLM 时显式传入 JSON Schema。

### 4.3 `paperinsight/llm/openai_client.py`

改造目标：
- 优先尝试 `json_schema + strict:true`；
- 若不支持，再回退为 `json_object`；
- 继续保留最终文本 JSON 兜底解析；
- 记录 usage，便于后续统计 Prompt Cache 命中信息。

### 4.4 `paperinsight/web/impact_factor_search.py`

改造目标：
- 修复旧版固定 URL + 粗 regex 解析不稳定的问题；
- 改为“搜索页 -> 候选详情页 -> 历史 IF 解析”的三段式策略；
- 增加候选期刊相似度匹配，减少串刊风险。

### 4.5 `paperinsight/cli.py`

改造目标：
- 在每次运行前确认是否开启双语输出；
- CLI 摘要中显示“双语输出：启用/禁用”；
- 该选择仅影响本次运行，不强制覆盖用户持久配置。

### 4.6 `paperinsight/core/reporter.py`

改造目标：
- 支持双语字段的分段导出；
- 当值符合以下格式时：
  - `中文：...`
  - `English: ...`
- Excel 单元格使用富文本：
  - 中文行加粗
  - 英文行常规显示

## 5. 预期收益

### 5.1 提取准确率提升
- 摘要中的 EQE、结果中的结构、图注中的补充参数、表格中的多器件数据能被同时保留下来。
- 多器件论文中，结构与性能错位的概率下降。

### 5.2 成本与稳定性提升
- 不再依赖章节白名单造成的“过保留/过裁剪”二选一。
- 在 cleaner 阶段完成主要压缩，减少 extractor 侧粗暴截断带来的信息丢失。

### 5.3 报表可读性提升
- 双语字段改为上下分段显示，明显优于 `中文 | English` 的单行拼接。
- 中文部分加粗后，阅读和核对更直观。

### 5.4 影响因子列可靠性提升
- 对缺失值和明显错误值提供自动联网校正能力。
- 解决当前版本影响因子获取异常导致的排序与重命名质量下降问题。

---

## 6. 验收标准

### 功能验收
1. 对包含摘要 EQE、正文结构、表格参数、图注说明的论文样本，v3.1 应保留这些信息进入 LLM 输入。
2. `References` 中的内容不得再进入最终提取上下文。
3. 影响因子为空时，应能联网补全；影响因子明显异常时，应能进行矫正。
4. 每次运行前必须出现双语确认。
5. 启用双语后，导出结果中的自然语言字段应为两行格式，不再使用 `|`。
6. Excel 中双语单元格的中文行需为加粗显示。

### 代码验收
1. `section_filter.py` 完成块级过滤重构。
2. `extractor.py` 去除固定 15000 字符硬截断主逻辑。
3. `openai_client.py` 支持严格 schema 输出优先。
4. `impact_factor_search.py` 完成新的搜索与解析逻辑。
5. CLI 增加本次运行的双语确认逻辑。
6. 报表支持富文本样式输出。

## 7. v3.1 范围说明

### 本版本必须完成
- 结构感知过滤
- 表格锚点化
- 窗口上下文保留
- 严格 schema 输出优先
- 影响因子联网校正
- 双语确认交互
- 双语分段 + 中文加粗导出

### 可放入 3.1.x 的增强项
- 多器件两阶段提取
- 图注/表注更细粒度分类
- Provider 级 Prompt Cache 命中统计落盘
- 基于标题路径的轻量 rerank
