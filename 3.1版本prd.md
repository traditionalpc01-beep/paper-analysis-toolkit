# 产品需求文档 (PRD): PaperInsight v3.1 (语义窗口过滤引擎版)

## 1. 产品概述与演进目标
**PaperInsight v3.1** 是在 v3.0（MinerU + LLM 嵌套提取）基础上的“精准度与成本双向优化”版本。
针对科研论文中数据分布高度离散（如参数散落在图注、正文各处）的痛点，v3.1 废弃了传统的“按章节名粗暴裁剪”方案，引入了业界领先的 **“Markdown 层级感知 + 实体密度滑动窗口（Entity-Density Sliding Window）”** 算法。
**目标**：在确保核心数据（如极端位置的 EQE、边缘设备结构）**0 遗漏**的前提下，将输入给 LLM 的 Token 量压缩 **60% - 80%**，极大降低 API 成本并消除大模型的“中间注意力丢失（Lost in the middle）”问题。

## 2. 核心架构设计 (The 3.1 Pipeline)

### 2.1 阶段一：Markdown 层级感知与绝对噪声剔除
MinerU 吐出的完整 Markdown 包含文档的天然树状结构。
*   **动作**：不再按白名单（如只留 Results）过滤，而是**按黑名单**进行极简切除。
*   **逻辑**：使用正则表达式或 Markdown 解析树，**仅**剥离文档末尾的绝对噪声章节：
    *   `References / Bibliography` (参考文献，极其消耗 Token 且易引发幻觉)
    *   `Acknowledgements` (致谢)
    *   `Author Contributions` (作者贡献)
    *   `Data Availability` (数据可用性声明)
*   **表格锚点化 (Table Anchoring)**：将所有 Markdown 表格 (`|...|...|`) 完整提取，并在原位置留下占位符 `[TABLE_X_INSERTED_HERE]`。表格内容享有**最高优先级，绝对保留**。

### 2.2 阶段二：领域词典与滑动窗口过滤 (核心创新) [INDEX_1.11]
这是 3.1 版本的灵魂。将清洗后的正文按段落（Paragraph，即 `\n\n`）或 Markdown 块切分。

*   **1. 构建光电领域实体词典 (Domain Lexicon)**：
    *   定义核心参数触发词：`["EQE", "CIE", "efficiency", "lifetime", "T50", "T90", "cd/A", "lm/W", "V", "mA/cm2"]`
    *   定义器件结构触发词：`["structure", "device", "ITO", "FTO", "layer", "spin-coating", "HTL", "ETL", "EML"]`
*   **2. 密度计算与命中 (Hit)**：
    *   遍历每一个段落。只要段落中包含上述词典中的**任意 1 个**词，该段落即被标记为 `Hit`（命中）。
*   **3. 滑动窗口保留 (Sliding Window Context)** [INDEX_1.11]：
    *   **痛点**：有时候段落 A 写了“器件结构”，段落 B 才写“它的 EQE 是 20%”。如果只保留段落 B，模型就不知道这是哪个器件的 EQE。
    *   **策略**：如果段落 $N$ 被标记为 `Hit`，程序不仅保留段落 $N$，还要**强制保留它的上下文段落：段落 $N-1$ 和 段落 $N+1$**。
    *   将这些合并后的窗口去重，形成高密度的“信息浓缩块”。

### 2.3 阶段三：Prompt 缓存 (Prompt Caching) 与嵌套提取 [INDEX_1.5]
*   **Context 重组**：将所有保留的滑动窗口文本，与之前提取的 Markdown 表格拼装在一起，生成最终的 `Condensed_Markdown`。
*   **API 交互标准 (2026)**：
    *   强制开启大模型 API 的 **Prompt Caching** 选项（如果使用的是 OpenAI/Claude/DeepSeek 最新的 API），这能在处理同系列格式文件时进一步降低 50% 成本[INDEX_1.5]。
    *   使用 **Structured Output (JSON Schema)**，强制要求 LLM 输出包含 `devices: [{...}]` 的嵌套数组，确保数据不错位。

class SmartWindowFilter:
    def __init__(self, window_size=1):
        self.window_size = window_size
        # 光电器件核心特征词典（全部小写）
        self.lexicon = {
            "eqe", "cie", "lifetime", "t50", "efficiency", 
            "cd/a", "lm/w", "ma/cm2", "device", "structure", 
            "ito", "fto", "pedot", "layer", "optimized"

    def clean(self, raw_md: str) -> str:
        # 1. 暴力切除尾部参考文献 (简单正则匹配最后一个 Reference 标题之后的所有内容)
        md_no_refs = self._remove_references(raw_md)
        
        # 2. 提取表格并赋予最高权重保留
        tables = self._extract_tables(md_no_refs)
        
        # 3. 按段落切分文本 (Markdown的段落通常由两个换行符分隔)
        paragraphs =[p.strip() for p in md_no_refs.split('\n\n') if p.strip()]
        
        # 4. 滑动窗口命中逻辑
        keep_indices = set()
        for i, para in enumerate(paragraphs):
            # 将段落转小写并检查是否包含关键词
            para_lower = para.lower()
            if any(keyword in para_lower for keyword in self.lexicon):
                # 如果命中，保留当前段落，以及前后 window_size 个段落
                start_idx = max(0, i - self.window_size)
                end_idx = min(len(paragraphs) - 1, i + self.window_size)
                for j in range(start_idx, end_idx + 1):
                    keep_indices.add(j)
                    
        # 5. 按照原顺序重组保留的段落

        condensed_text = ""
        for idx in sorted_indices:

        # 6. 最终组装送给 LLM 的 Payload

{tables}

{condensed_text}
