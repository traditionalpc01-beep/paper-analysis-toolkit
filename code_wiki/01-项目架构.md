# PaperInsight 项目架构文档

## 项目概述

**PaperInsight** 是一个智能科研论文分析工具，专注于从 PDF 格式的学术论文中提取结构化数据。项目采用 Python 开发，支持命令行界面（CLI）和桌面应用（Electron + React）两种使用方式。

- **当前版本**: 4.1.0
- **开发语言**: Python 3.9+
- **许可证**: MIT
- **项目地址**: https://github.com/traditionalpc01-beep/paper-analysis-toolkit

## 核心处理流程

### 分析管线流程

```
PDF 输入
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 1: PDF 解析                                            │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ MinerU API/CLI → Markdown + 表格 + 元数据               │ │
│ │ 失败时回退 → PyMuPDF/pdfplumber 基础解析                │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 2: 文本清洗                                            │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ SectionFilter → 移除噪声章节（参考文献、致谢等）        │ │
│ │ → 保留核心章节（摘要、实验、结果）                      │ │
│ │ → 提取表格和图注                                        │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 数据提取                                            │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ LLM 模式 → 结构化 JSON 提取（支持多 LLM 厂商）          │ │
│ │ Regex 模式 → 正则表达式兜底提取                         │ │
│ │ Pydantic 校验 → 数据验证和清洗                          │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 4: 数据补全                                            │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 期刊元数据解析 → MJL Journal Resolver                   │ │
│ │ 影响因子获取 → LetPub + MJL + WOS + AI Model            │ │
│ │ 数据验证和交叉校验                                      │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Step 5: 报告生成                                            │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ Excel 报告 → openpyxl 生成                              │ │
│ │ JSON 报告 → 结构化数据导出                              │ │
│ │ 错误日志 → 失败记录追踪                                 │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
    ↓
输出文件 (Excel/JSON)
```

## 双模式架构

项目支持两种运行模式：

### API 模式（推荐）
- 使用 LLM 进行语义化数据提取
- 调用在线 OCR 服务（MinerU API、PaddleX）
- 自动获取期刊影响因子
- 支持多数据源交叉验证

### Regex 模式（兜底）
- 纯本地正则表达式提取
- 无需网络连接
- 基础 PDF 解析（PyMuPDF）
- 适用于离线环境或 API 不可用场景

## 缓存机制

```
缓存目录结构:
.cache/
├── <md5>_data.json      # 提取结果缓存
├── <md5>_markdown.md    # PDF 解析结果缓存
└── <md5>_ocr.md         # OCR 结果缓存（兼容旧版）

缓存策略:
- 基于 PDF 文件 MD5 指纹
- 支持缓存命中检测
- 可通过 --no-cache 禁用
- 支持缓存清理命令
```

## 配置系统

```
配置文件位置: ~/.paperinsight/config.yaml

配置层级:
1. 默认配置 (utils/config.py::DEFAULT_CONFIG)
2. 用户配置文件 (config.yaml)
3. 命令行参数覆盖

敏感信息处理:
- API Key 等敏感字段加密存储
- 使用 XOR + Base64 混淆
- 文件权限设置为 0600
```

## 性能优化

1. **并行处理**: 使用线程池并行处理多个 PDF
2. **批量 API**: MinerU 支持批量解析，减少 API 调用
3. **智能缓存**: 避免重复解析和提取
4. **增量处理**: 仅处理新增或修改的文件
5. **内存优化**: 流式处理大型 PDF

## 安全设计

1. **配置加密**: 敏感配置项加密存储
2. **权限控制**: 配置文件权限限制
3. **输入验证**: Pydantic 模型校验
4. **错误隔离**: 单文件失败不影响批量处理
5. **日志脱敏**: 不记录敏感信息
