# PaperInsight 模块职责文档

本文档详细说明 PaperInsight 项目各模块的职责、功能和相互关系。

## 模块总览

```
paperinsight/
├── core/          # 核心处理模块 - 分析管线和数据提取
├── parser/        # 解析器模块 - PDF 文档解析
├── cleaner/       # 清洗器模块 - 文本预处理和噪声过滤
├── llm/           # LLM 模块 - 大语言模型集成
├── models/        # 数据模型 - 结构化数据定义
├── ocr/           # OCR 模块 - 光学字符识别
├── web/           # Web 服务模块 - 在线数据获取
└── utils/         # 工具模块 - 通用工具函数
```

---

## 1. 核心模块 (core/)

### 1.1 pipeline.py - 分析管线

**职责**: 协调整个论文分析流程，是系统的核心调度器。

**主要功能**:
- PDF 文件批量处理
- 解析、清洗、提取流程编排
- 缓存管理
- 报告生成调度
- 错误处理和日志记录

**核心类**:
- `AnalysisPipeline`: 主分析管线类

**关键方法**:
| 方法 | 功能 |
|------|------|
| `process_pdf()` | 处理单个 PDF 文件 |
| `process_batch()` | 批量处理 PDF 文件 |
| `run()` | 运行完整分析流程 |
| `_parse_pdf()` | PDF 解析调度 |
| `_supplement_impact_factor()` | 影响因子补全 |

**依赖关系**:
```
AnalysisPipeline
    ├── CacheManager (缓存)
    ├── MinerUParser (解析)
    ├── SectionFilter (清洗)
    ├── DataExtractor (提取)
    ├── ReportGenerator (报告)
    └── Web 服务 (数据补全)
```

### 1.2 extractor.py - 数据提取器

**职责**: 从清洗后的文本中提取结构化数据。

**主要功能**:
- LLM 语义化数据提取
- 正则表达式兜底提取
- 数据验证和清洗
- 多器件信息提取
- 期刊元数据提取

**核心类**:
- `DataExtractor`: 数据提取器主类

**关键方法**:
| 方法 | 功能 |
|------|------|
| `extract()` | 主提取入口 |
| `_extract_with_llm()` | LLM 模式提取 |
| `_extract_with_regex()` | 正则模式提取 |
| `_extract_title()` | 提取论文标题 |
| `_extract_authors()` | 提取作者信息 |
| `_extract_devices()` | 提取器件数据 |
| `_extract_raw_journal_metadata()` | 提取期刊元数据 |

**提取字段**:
- 论文基本信息：标题、作者、期刊、年份
- 器件性能参数：EQE、CIE、寿命、亮度
- 器件结构：各层材料和厚度
- 优化策略：优化层级和具体方法

### 1.3 cache.py - 缓存管理

**职责**: 管理解析结果和提取结果的缓存。

**主要功能**:
- 基于 MD5 的缓存键生成
- 缓存读写操作
- 缓存统计和清理

**核心类**:
- `CacheManager`: 缓存管理器

**缓存类型**:
| 缓存类型 | 文件格式 | 内容 |
|----------|----------|------|
| 数据缓存 | `<md5>_data.json` | 提取的结构化数据 |
| Markdown 缓存 | `<md5>_markdown.md` | PDF 解析结果 |
| OCR 缓存 | `<md5>_ocr.md` | OCR 结果（兼容旧版） |

### 1.4 reporter.py - 报告生成器

**职责**: 生成分析报告（Excel/JSON 格式）。

**主要功能**:
- Excel 报告生成
- JSON 报告生成
- 数据排序和格式化
- 错误日志记录

**核心类**:
- `ReportGenerator`: 报告生成器
- `ErrorLogger`: 错误日志记录器

**输出格式**:
- Excel: 20 列固定格式，支持多器件换行显示
- JSON: 完整结构化数据，包含所有嵌套信息

## 2. 解析器模块 (parser/)

### 2.1 base.py - 解析器基类

**职责**: 定义文档解析器的统一接口。

**核心类**:
- `BaseParser`: 解析器抽象基类
- `ParseResult`: 解析结果数据结构

**关键方法**:

|------|------|
| `parse()` | 解析文档（抽象方法） |
| `is_available()` | 检查解析器可用性 |

**职责**: 使用 MinerU 服务解析 PDF 文档。

**主要功能**:
- MinerU API 调用
- MinerU CLI 调用
- 批量解析支持
- 结果格式转换

**核心类**:
- `MinerUParser`: MinerU 解析器实现

**运行模式**:
| 模式 | 说明 |
|------|------|
| `api` | 调用 MinerU 云端 API |
| `cli` | 调用本地 MinerU CLI |

**输出内容**:
- Markdown 格式文本
- 表格数据
- 文档元数据
- 章节结构

## 3. 清洗器模块 (cleaner/)

### 3.1 section_filter.py - 章节过滤器

**职责**: 过滤和压缩 PDF 解析后的文本内容。

**主要功能**:
- 噪声章节识别和移除
- 核心章节提取
- 表格锚点处理
- 文本压缩和预算控制

**核心类**:
- `SectionFilter`: 章节过滤器
- `CleanedContent`: 清洗结果结构
- `MarkdownBlock`: Markdown 块结构

**过滤策略**:
```
噪声章节（移除）:
- 参考文献 (References)
- 致谢 (Acknowledgments)
- 补充材料 (Supplementary)
- 利益冲突声明
- 作者贡献

核心章节（保留）:
- 摘要 (Abstract)
- 引言 (Introduction)
- 实验 (Experimental)
- 结果 (Results)
- 讨论 (Discussion)
```

**评分机制**:
- 参数关键词命中：+2.5 分
- 结构关键词命中：+2.0 分
- 数值指标命中：+3.0 分
- 图注内容：+2.0 分
- 噪声章节：-6.0 分

## 4. LLM 模块 (llm/)

### 4.1 base.py - LLM 基类

**职责**: 定义 LLM 客户端的统一接口。

**核心类**:
- `BaseLLM`: LLM 客户端抽象基类

**关键方法**:
| 方法 | 功能 |
|------|------|
| `generate()` | 生成文本 |
| `generate_json()` | 生成 JSON 格式输出 |
| `is_available()` | 检查服务可用性 |

### 4.2 各厂商客户端

| 文件 | 提供商 | 特点 |
|------|--------|------|
| `openai_client.py` | OpenAI | 支持 GPT-4o，JSON Schema |
| `deepseek_client.py` | DeepSeek | 国产大模型，性价比高 |
| `wenxin_client.py` | 百度文心 | 需要client_id和client_secret |
| `longcat_client.py` | Longcat | 支持长文本，Lite 模式回填 |

### 4.3 prompt_templates.py - Prompt 模板

**职责**: 管理 LLM 调用的 Prompt 模板。

**主要模板**:
- `format_extraction_prompt_v3()`: 主提取 Prompt
- `format_lite_paper_info_backfill_prompt()`: 轻量回填 Prompt
- `format_bilingual_postprocess_prompt()`: 双语处理 Prompt

## 5. 数据模型模块 (models/)

### 5.1 schemas.py - Pydantic 模型

**职责**: 定义结构化数据模型和数据验证。

**核心模型**:

```
PaperData (完整论文数据)
├── paper_info: PaperInfo        # 论文基本信息
│   ├── title                    # 标题
│   ├── authors                  # 作者
│   ├── journal_name             # 期刊名称
│   ├── impact_factor            # 影响因子
│   ├── year                     # 年份
│   └── ...
├── devices: List[DeviceData]    # 器件数据列表
│   ├── device_label             # 器件标签
│   ├── structure                # 器件结构
│   ├── eqe                      # 外量子效率
│   ├── cie                      # CIE 坐标
│   ├── lifetime                 # 寿命
│   └── ...
├── data_source: DataSourceReference  # 数据溯源
│   ├── eqe_source               # EQE 原文
│   ├── cie_source               # CIE 原文
│   └── ...
└── optimization: OptimizationInfo     # 优化策略
    ├── level                    # 优化层级
    ├── strategy                 # 具体策略
    └── key_findings             # 关键发现
```

**验证规则**:
- 字符串清理（移除多余空白）
- 数值范围检查（影响因子 0.1-200）
- 必填字段检查

## 6. OCR 模块 (ocr/)

### 6.1 base.py - OCR 基类

**职责**: 定义 OCR 引擎的统一接口。

**核心类**:
- `BaseOCR`: OCR 引擎抽象基类

### 6.2 local.py - 本地 OCR

**职责**: 使用本地 PaddleOCR 进行文字识别。

**适用场景**:
- 离线环境
- 扫描版 PDF
- 图片型 PDF

### 6.3 paddlex_api.py - PaddleX API

**职责**: 调用 PaddleX 云端 OCR 服务。

**优势**:
- 更高的识别准确率
- 支持复杂版面
- 无需本地部署

## 7. Web 服务模块 (web/)

### 7.1 journal_resolver.py - 期刊解析器

**职责**: 解析期刊元数据，获取标准期刊名称和 ISSN。

**数据源**: MJL (Master Journal List)

**核心类**:
- `MJLJournalResolver`: 期刊解析器
- `MJLJournalResolution`: 解析结果

**解析方式**:
- 期刊标题匹配
- ISSN 匹配
- 模糊搜索

### 7.2 impact_factor_fetcher.py - 影响因子获取

**职责**: 从 MJL 获取官方影响因子。

**核心类**:
- `MJLImpactFactorFetcher`: 影响因子获取器

### 7.3 letpub_fetcher.py - LetPub 数据源

**职责**: 从 LetPub 获取影响因子和期刊信息。

**特点**:
- 数据更新及时
- 包含中科院分区
- 支持中文期刊名

### 7.4 wos_journal_fetcher.py - WOS API

**职责**: 从 Web of Science 获取期刊数据。

**特点**:
- 官方权威数据
- 需要 API Key
- 数据最准确

### 7.5 ai_model_if_fetcher.py - AI 模型 IF 获取

**职责**: 使用 AI 模型获取影响因子（实验性功能）。

**支持的 AI 模型**:
- 通义千问 (Qianwen)
- Kimi

### 7.6 search_crawler_fetcher.py - 搜索爬虫

**职责**: 通过搜索引擎获取影响因子信息。

**适用场景**:
- 其他数据源无结果时
- 新期刊信息获取

## 8. 工具模块 (utils/)

### 8.1 config.py - 配置管理

**职责**: 加载和管理用户配置。

**配置层级**:
1. 默认配置 (`DEFAULT_CONFIG`)
2. 用户配置文件 (`~/.paperinsight/config.yaml`)
3. 环境变量覆盖

**核心函数**:
- `load_config()`: 加载配置
- `get_config_path()`: 获取配置文件路径

### 8.2 config_crypto.py - 配置加密

**职责**: 加密和解密敏感配置项。

**加密方式**:
- 本地密钥生成
- XOR 混淆
- Base64 编码

### 8.3 config_wizard.py - 配置向导

**职责**: 交互式配置引导。

**配置流程**:
1. LLM 提供商选择
2. API Key 配置
3. MinerU 配置
4. OCR 配置

### 8.4 logger.py - 日志系统

**职责**: 统一日志管理。

**日志级别**:
- DEBUG: 调试信息
- INFO: 常规信息
- WARNING: 警告信息
- ERROR: 错误信息

### 8.5 pdf_utils.py - PDF 工具

**职责**: PDF 文件处理工具函数。

**主要功能**:
- PDF 元数据提取
- 文本提取
- 页面处理

### 8.6 file_renamer.py - 文件重命名

**职责**: 根据提取结果重命名 PDF 文件。

**命名模板**:
```
[{year}_{impact_factor}_{journal}]_{title}.pdf
```

### 8.7 hash_utils.py - 哈希工具

**职责**: 计算文件哈希值（用于缓存键）。

### 8.8 journal_metadata.py - 期刊元数据

**职责**: 期刊名称标准化和别名处理。

### 8.9 terminal.py - 终端工具

**职责**: 终端输出格式化和美化。

### 8.10 env_checker.py - 环境检查

**职责**: 检查运行环境和依赖。

## 9. 命令行接口 (cli.py)

**职责**: 提供命令行交互入口。

**命令列表**:

| 命令 | 功能 |
|------|------|
| `analyze` | 分析 PDF 文件 |
| `config` | 运行配置向导 |
| `check` | 环境检查 |
| `doctor` | 详细诊断 |
| `version` | 显示版本 |
| `cache-info` | 缓存统计 |
| `clear-cache` | 清理缓存 |

**analyze 命令参数**:

| 参数 | 说明 |
|------|------|
| `--mode` | 运行模式 (auto/api/regex) |
| `--output` | 输出目录 |
| `--recursive` | 递归扫描 |
| `--json` | 导出 JSON |
| `--bilingual` | 双语输出 |
| `--no-cache` | 禁用缓存 |
| `--rename-pdfs` | 重命名 PDF |
