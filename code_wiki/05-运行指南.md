# PaperInsight 运行指南文档

本文档详细说明 PaperInsight 项目的安装、配置和运行方式。

---

## 1. 环境要求

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10+, macOS 10.15+, Linux |
| Python | 3.9 - 3.12 |
| 内存 | 最低 4GB，推荐 8GB+ |
| 磁盘空间 | 最低 500MB（不含缓存） |

### 1.2 网络要求

| 功能 | 网络需求 |
|------|----------|
| LLM 提取 | 需要访问 LLM API（OpenAI/DeepSeek/文心/Longcat） |
| MinerU API | 需要访问 MinerU 云服务 |
| 影响因子获取 | 需要访问 MJL/LetPub/WOS 等 API |
| Regex 模式 | 无需网络（纯本地运行） |

## 2. 安装方式

### 2.1 从源码安装（推荐）

```bash
# 克隆仓库
git clone https://github.com/traditionalpc01-beep/paper-analysis-toolkit.git
cd paper-analysis-toolkit

# 创建虚拟环境（推荐）
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 安装基础依赖
pip install -r requirements.txt

# 安装项目
pip install -e .
```

### 2.2 安装可选依赖

```bash
# LLM 支持
pip install openai>=1.0.0

# OCR 支持
pip install paddlepaddle>=3.0.0 paddleocr>=3.0.0

# 开发工具
pip install pytest>=7.0.0 black>=23.0.0

# 或使用 extras
pip install -e ".[llm,ocr,dev]"
```

### 2.3 验证安装

```bash
# 检查版本
paperinsight version

# 环境检查
paperinsight check
```

## 3. 配置

### 3.1 配置文件位置

```
~/.paperinsight/config.yaml
```

### 3.2 配置向导

首次使用建议运行配置向导：

```bash
paperinsight config
```

配置向导会引导配置：
1. LLM 提供商选择和 API Key 配置
2. MinerU 配置（API Token 或 CLI 路径）
3. OCR 配置（可选）
4. Web 搜索配置

### 3.3 手动配置

创建配置文件 `~/.paperinsight/config.yaml`：

```yaml
# LLM 配置
llm:
  enabled: true
  provider: openai          # openai, deepseek, wenxin, longcat
  api_key: sk-xxxxx
  model: gpt-4o
  timeout: 120
  base_url: null            # 可选，用于代理
  
  # 文心一言专用配置
  wenxin:
    client_id: xxxx
    client_secret: xxxx
  
  # Longcat 专用配置
  longcat:
    model: LongCat-Flash-Chat
    base_url: https://api.longcat.chat/openai
    enable_lite_backfill: true
    backfill_model: LongCat-Flash-Lite

# MinerU 配置
mineru:
  enabled: true
  mode: api                 # api 或 cli
  token: your-mineru-token
  cli_path: null            # CLI 模式需要

# PaddleX OCR 配置
paddlex:
  enabled: false
  token: null

# 缓存配置
cache:
  enabled: true
  directory: .cache

# 输出配置
output:
  format:
    - excel                 # excel, json
  directory: null           # 默认为 PDF 目录下的 输出结果/
  sort_by_if: true
  rename_pdfs: false
  rename_template: "[{year}_{impact_factor}_{journal}]_{title}.pdf"
  bilingual_text: false

# 文本清洗配置
cleaner:
  enabled: true
  max_input_chars: 24000
  max_blocks: 80
  min_block_score: 3.0
  block_window: 1

# Web 搜索配置
web_search:
  enabled: true
  timeout: 30
  resolve_journal_metadata: true
  fetch_official_impact_factor: true
  correct_existing_impact_factor: true
  
  letpub:
    enabled: true
    timeout: 30
  
  search_crawler:
    enabled: true
    market: en-US
  
  web_of_science:
    enabled: false
    api_key: null
  
  ai_model_if:
    enabled: false
    timeout: 30
    qianwen_api_key: null
    kimi_api_key: null

# PDF 配置
pdf:
  max_pages: null
  text_ratio_threshold: 0.1
```

### 3.4 环境变量配置

也可以通过环境变量配置：

```bash
# Linux/Mac
export OPENAI_API_KEY=sk-xxxxx
export MINERU_TOKEN=your-token

# Windows
set OPENAI_API_KEY=sk-xxxxx
set MINERU_TOKEN=your-token
```

## 4. 命令行使用

### 4.1 基本命令

```bash
# 显示帮助
paperinsight --help

# 显示版本
paperinsight version

# 环境检查
paperinsight check

# 详细诊断
paperinsight doctor

# 配置向导
paperinsight config

# 缓存信息
paperinsight cache-info

# 清理缓存
paperinsight clear-cache
```

### 4.2 analyze 命令

#### 基本用法

```bash
# 分析单个目录
paperinsight analyze ./pdfs

# 指定输出目录
paperinsight analyze ./pdfs --output ./results

# 递归扫描子目录
paperinsight analyze ./pdfs --recursive
```

#### 模式选择

```bash
# 自动选择模式（推荐）
paperinsight analyze ./pdfs --mode auto

# 强制 API 模式
paperinsight analyze ./pdfs --mode api

# 强制 Regex 模式（离线）
paperinsight analyze ./pdfs --mode regex
```

#### 输出选项

```bash
# 同时导出 JSON
paperinsight analyze ./pdfs --json

# 双语输出（中英对照）
paperinsight analyze ./pdfs --bilingual

# 按影响因子排序
paperinsight analyze ./pdfs --sort-by-if
```

#### 缓存控制

```bash
# 使用缓存（默认）
paperinsight analyze ./pdfs

# 禁用缓存，重新处理
paperinsight analyze ./pdfs --no-cache
```

#### 文件重命名

```bash
# 分析后重命名 PDF
paperinsight analyze ./pdfs --rename-pdfs

# 自定义命名模板
paperinsight analyze ./pdfs --rename-pdfs
# 默认模板: [{year}_{impact_factor}_{journal}]_{title}.pdf
```

#### 其他选项

```bash
# 限制最大页数
paperinsight analyze ./pdfs --max-pages 10

# 跳过启动检查
paperinsight analyze ./pdfs --skip-checks
```

### 4.3 完整参数说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `pdf_dir` | - | PDF 目录 | 必填 |
| `--output` | `-o` | 输出目录 | PDF 目录/output |
| `--recursive` | `-r` | 递归扫描 | false |
| `--max-pages` | - | 最大页数 | 无限制 |
| `--mode` | `-m` | 运行模式 | auto |
| `--no-cache` | - | 禁用缓存 | false |
| `--json` | - | 导出 JSON | false |
| `--bilingual` | - | 双语输出 | false |
| `--rename-pdfs` | - | 重命名 PDF | false |
| `--skip-checks` | - | 跳过检查 | false |

## 5. Python API 使用

### 5.1 基本使用

```python
from paperinsight.core.pipeline import AnalysisPipeline

# 创建管线
pipeline = AnalysisPipeline(
    output_dir="./output",
    config={
        "llm": {
            "enabled": True,
            "provider": "openai",
            "api_key": "sk-xxxxx",
        },
        "mineru": {
            "enabled": True,
            "mode": "api",
            "token": "your-token",
        },
    },
)

# 运行分析
stats = pipeline.run(
    pdf_dir="./pdfs",
    recursive=True,
    use_cache=True,
)

print(f"成功: {stats['success_count']}")
print(f"失败: {stats['error_count']}")
print(f"报告: {stats['report_files']}")
```

### 5.2 处理单个文件

```python
from pathlib import Path
from paperinsight.core.pipeline import AnalysisPipeline

pipeline = AnalysisPipeline(output_dir="./output")

# 处理单个 PDF
paper_data, error = pipeline.process_pdf(
    pdf_path=Path("./paper.pdf"),
    use_cache=True,
)

if paper_data:
    print(f"标题: {paper_data.paper_info.title}")
    print(f"期刊: {paper_data.paper_info.journal_name}")
    print(f"影响因子: {paper_data.paper_info.impact_factor}")
    
    for device in paper_data.devices:
        print(f"器件: {device.device_label}")
        print(f"EQE: {device.eqe}")
```

### 5.3 数据提取

```python
from paperinsight.core.extractor import DataExtractor

# 创建提取器
extractor = DataExtractor(
    config={
        "llm": {
            "enabled": True,
            "provider": "openai",
            "api_key": "sk-xxxxx",
        }
    }
)

# 提取数据
result = extractor.extract(
    markdown_text=markdown_content,
    cleaned_text=cleaned_content,
)

if result.success:
    print(f"提取方法: {result.extraction_method}")
    print(f"处理时间: {result.processing_time:.2f}s")
```

### 5.4 文本清洗

```python
from paperinsight.cleaner.section_filter import SectionFilter

# 创建过滤器
filter = SectionFilter(config={
    "enabled": True,
    "max_input_chars": 24000,
})

# 清洗内容
result = filter.clean(markdown_text)

print(f"压缩率: {result.reduction_ratio:.2%}")
print(f"摘要长度: {len(result.abstract)}")
print(f"实验部分长度: {len(result.experimental)}")
```

## 9. 故障排除

### 9.1 环境问题

```bash
# 运行诊断
paperinsight doctor

# 检查 Python 版本
python --version

# 检查依赖
pip list | grep -E "typer|rich|PyMuPDF"
```

### 9.2 配置问题

```bash
# 重新配置
paperinsight config

# 检查配置文件
cat ~/.paperinsight/config.yaml
```

### 9.3 API 问题

```bash
# 测试 LLM 连接
paperinsight check

# 检查 API Key 是否有效
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

### 9.4 缓存问题

```bash
# 查看缓存状态
paperinsight cache-info

# 清理缓存
paperinsight clear-cache
```

### 9.5 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `ModuleNotFoundError` | 依赖未安装 | `pip install -r requirements.txt` |
| `API Key invalid` | API Key 无效 | 检查配置文件中的 API Key |
| `MinerU unavailable` | MinerU 不可用 | 检查 Token 或 CLI 安装 |
| `No PDF files found` | 目录无 PDF | 检查目录路径和文件扩展名 |
| `Extraction failed` | 提取失败 | 检查 LLM 配置或使用 Regex 模式 |
