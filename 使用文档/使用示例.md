# 使用示例

下面的例子都基于当前 CLI：`paperinsight`。

## 场景一：先离线试跑

适合第一次接触项目，先确认目录、输出和字段格式。

```bash
paperinsight analyze ./pdfs --mode regex
```

适用特点：

- 不依赖在线 OCR 或 LLM
- 速度快
- 适合先看报表结构是否符合预期

## 场景二：目录里有很多子文件夹

```bash
paperinsight analyze ./pdfs --recursive
```

效果：

- 会递归扫描所有子目录中的 `.pdf`
- 输出仍统一写入当前分析目录下的 `输出结果/`

## 场景三：开启智能提取

先配置：

```bash
paperinsight config
```

再运行：

```bash
paperinsight analyze ./pdfs --mode api
```

适用特点：

- 优先使用在线 OCR / LLM
- 对扫描版、复杂版式和语义提取更友好
- 适合正式批量处理

## 场景四：同时保留 JSON

```bash
paperinsight analyze ./pdfs --json
```

适合：

- 后续要写 Python / pandas 脚本做统计
- 想把结构化结果接到别的流程中

## 场景五：处理完成后重命名 PDF

```bash
paperinsight analyze ./pdfs --rename-pdfs
```

默认命名模板类似：

```text
[2024_IF12.3_NatCommun]_Paper_Title.pdf
```

适合：

- 做文献归档
- 后续按年份、期刊和 IF 直接浏览文件夹

## 场景六：限制读取页数做快速预览

```bash
paperinsight analyze ./pdfs --max-pages 5
```

适合：

- 先做快速抽检
- 在大量论文上验证配置是否合理

## 场景七：查看环境诊断

```bash
paperinsight doctor
```

你会看到：

- Python 版本
- 网络状态
- PaddleX / LLM 配置状态
- 本地 OCR 状态
- 推荐的运行模式

## 场景八：查看缓存状态

```bash
paperinsight cache-info
```

适合：

- 判断是否已经有大量缓存可复用
- 排查为什么第二次运行明显变快

## 场景九：清空缓存重新跑

```bash
paperinsight clear-cache
paperinsight analyze ./pdfs --mode api
```

适合：

- 修改了提取逻辑后重跑
- 想重新生成 OCR 文本和结构化结果

## 场景十：用 pandas 读取分析结果

```python
import pandas as pd

report = pd.read_excel("./pdfs/输出结果/Paper_Analysis_Report.xlsx")
print(report[["期刊名称", "影响因子", "论文标题"]].head())
```

如果你导出了 JSON：

```python
import json
from pathlib import Path

path = Path("./pdfs/输出结果/Paper_Analysis_Report.json")
data = json.loads(path.read_text(encoding="utf-8"))
print(data[0])
```
