# 使用示例

下面的示例只使用当前 CLI 已实现的参数与命令，不包含仓库中不存在的选项。

## 场景 1：第一次先离线试跑

```bash
paperinsight analyze ./pdfs --mode regex
```

适合：

- 先确认目录、报表结构、字段命名
- 暂时没有任何 API Key
- 想避免联网调用成本

## 场景 2：按当前配置自动选模式

```bash
paperinsight analyze ./pdfs
```

说明：

- 若检测到在线能力，`auto` 会落到 `api`
- 否则会落到 `regex`

## 场景 3：递归处理多层目录

```bash
paperinsight analyze ./pdfs --recursive
```

## 场景 4：同时导出 JSON

```bash
paperinsight analyze ./pdfs --json
```

当前 JSON 文件名和 Excel 一样，都是 `论文分析报告_<时间戳>.*`。

## 场景 5：关闭缓存重新跑

```bash
paperinsight analyze ./pdfs --no-cache
```

适合：

- 刚改了提取逻辑
- 想忽略已有 `_data.json` / `_markdown.md` 缓存

## 场景 6：处理完成后重命名 PDF

```bash
paperinsight analyze ./pdfs --rename-pdfs
```

默认模板来自配置：

```text
[{year}_{impact_factor}_{journal}]_{title}.pdf
```

## 场景 7：强制 API 模式

```bash
paperinsight analyze ./pdfs --mode api
```

适合：

- 已经配好 LLM / MinerU / 网络能力
- 想明确避免落到 `regex` 模式

## 场景 8：启用双语输出

```bash
paperinsight analyze ./pdfs --bilingual
```

说明：

- 只有在 LLM 可用时，这个选项才有意义。
- 未显式传参时，CLI 会在运行前询问一次是否启用双语输出。

## 场景 9：查看环境与缓存状态

```bash
paperinsight check
paperinsight doctor
paperinsight cache-info
```

如果要清空缓存：

```bash
paperinsight clear-cache
```

## 场景 10：开发模式启动桌面壳

```bash
cd desktop
npm install
npm run dev
```

## 场景 11：实验性 AI 影响因子补全

先把下面的键手工加入 `~/.paperinsight/config.yaml`：

```yaml
web_search:
  enabled: true
  use_ai_model_if: true
  ai_model_if:
    enabled: true
    timeout: 60
    qianwen_api_key: ""
    kimi_api_key: ""
```

然后运行：

```bash
paperinsight analyze ./pdfs --mode api
```

补充说明：

- 这条路径当前是实验性的。
- 当前实现依赖 `beautifulsoup4`；如果你是按 `requirements.txt` 或 `pyproject.toml` 安装，一般已经包含它。

## 场景 12：用 Python 读取报表

```python
import pandas as pd

report = pd.read_excel('./pdfs/输出结果/论文分析报告_20260318_153000.xlsx')
print(report[['期刊名称 Journal', '影响因子 Impact Factor', '论文标题 Title']].head())
```

如果你想按最新生成的文件自动读取，请先在文件系统中获取实际文件名；当前实现不会固定输出为 `Paper_Analysis_Report.xlsx`。
