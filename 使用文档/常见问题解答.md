# 常见问题解答

## 1. 运行 `paperinsight` 后看到的是引导，不是直接分析？

这是预期行为。

不带子命令启动时，CLI 会先显示：

- 启动引导
- 推荐命令顺序
- 当前配置状态
- 命令帮助

正式分析请运行：

```bash
paperinsight analyze ./pdfs
```

## 2. 首次运行应该先做什么？

建议顺序：

```bash
paperinsight check
paperinsight config
paperinsight analyze ./pdfs
```

如果你暂时没有 API Key，也可以直接：

```bash
paperinsight analyze ./pdfs --mode regex
```

## 3. `api` 模式为什么会进入配置向导？

当你运行：

```bash
paperinsight analyze ./pdfs --mode api
```

但还没有配置任何在线 OCR / LLM 能力时，CLI 会自动进入配置向导，避免你在“想走 API 模式”的情况下实际却落到基础模式。

## 4. 没有 API Key 还能用吗？

可以。

你仍然可以使用：

```bash
paperinsight analyze ./pdfs
```

或者：

```bash
paperinsight analyze ./pdfs --mode regex
```

只是对扫描版 PDF、复杂版式和语义提取的支持会弱一些。

## 5. 扫描版 PDF 提取效果差怎么办？

优先顺序建议是：

1. 配置 PaddleX
2. 安装本地 PaddleOCR 作为备用
3. 再重新运行分析

相关命令：

```bash
paperinsight config
paperinsight doctor
```

## 6. 为什么第二次运行快很多？

因为默认启用了缓存，会复用：

- OCR 文本缓存
- 结构化结果缓存

查看缓存：

```bash
paperinsight cache-info
```

清空缓存：

```bash
paperinsight clear-cache
```

## 7. 报表文件生成在哪？

默认在输入目录下的 `输出结果/` 中，例如：

```text
./pdfs/输出结果/Paper_Analysis_Report.xlsx
```

如果运行时使用了 `--json`，还会生成：

```text
./pdfs/输出结果/Paper_Analysis_Report.json
```

如果有失败项，会生成：

```text
./pdfs/输出结果/error_log.txt
```

## 8. 为什么有些论文没有提取到 EQE / CIE / 寿命？

常见原因：

- 文章本身不是器件结果导向
- PDF 是扫描版，但没有可用 OCR
- 数据表述方式不在当前规则覆盖范围内
- LLM 未启用，正则模式无法理解更复杂的表述

建议排查：

```bash
paperinsight doctor
paperinsight analyze ./pdfs --mode api
```

## 9. 影响因子是怎么来的？

程序当前按这个顺序尝试：

1. 论文首页或正文中显式出现的 IF / Impact Factor
2. Web 搜索补全（如果已启用）

如果你希望完全离线运行，可以在配置里关闭 `web_search.enabled`。

## 10. 如何重命名处理后的 PDF？

临时启用：

```bash
paperinsight analyze ./pdfs --rename-pdfs
```

长期启用：

在 `~/.paperinsight/config.yaml` 中设置：

```yaml
output:
  rename_pdfs: true
```

如果想自定义模板：

```yaml
output:
  rename_template: "[{year}_{impact_factor}_{journal}]_{title}.pdf"
```

## 11. 如何确认当前配置是否正确？

最直接的方法：

```bash
paperinsight doctor
```

它会展示：

- OCR 配置状态
- PaddleX 配置状态
- LLM 配置状态
- 本地 OCR 状态
- 推荐运行模式

## 12. Mac 上提示 `python` 或 `paperinsight` 找不到怎么办？

可以尝试：

```bash
python3 -m paperinsight.cli
python3 -m paperinsight.cli analyze ./pdfs
```

如果 `paperinsight` 命令不可用，通常是因为还没执行：

```bash
pip install -e .
```
