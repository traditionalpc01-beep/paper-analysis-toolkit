# 常见问题解答

## 1. 直接运行 `paperinsight` 为什么不是立刻分析？

这是当前设计。

不带子命令启动时，`paperinsight/cli.py` 的回调会先显示欢迎信息、推荐流程和帮助文本；真正执行分析需要显式运行：

```bash
paperinsight analyze ./pdfs
```

## 2. 首次运行建议先做什么？

推荐顺序：

```bash
paperinsight check
paperinsight config
paperinsight analyze ./pdfs
```

## 3. 没有 API Key 还能用吗？

可以。

```bash
paperinsight analyze ./pdfs --mode regex
```

当前 `regex` 模式是本地兜底模式，不依赖 LLM、MinerU API 或 WOS API。

## 4. `auto`、`api`、`regex` 三种模式有什么区别？

- `auto`：根据当前配置自动选择 `api` 或 `regex`
- `api`：尽量启用 LLM / 联网 / OCR 能力
- `regex`：关闭 LLM 和 PaddleX，走本地正则兜底

## 5. 输出文件生成在哪？为什么文件名不是固定的？

默认输出到输入目录下的 `输出结果/`，例如：

```text
./pdfs/输出结果/
```

当前 `ReportGenerator` 默认文件名是：

- `论文分析报告_<时间戳>.xlsx`
- `论文分析报告_<时间戳>.json`

不是固定的 `Paper_Analysis_Report.xlsx`。

## 6. 为什么第二次运行通常更快？

因为默认启用了缓存，会复用：

- `<md5>_data.json`
- `<md5>_markdown.md`
- 兼容旧版 `<md5>_ocr.md`

查看缓存：

```bash
paperinsight cache-info
```

清空缓存：

```bash
paperinsight clear-cache
```

## 7. 当前影响因子是怎么补全的？

当前可核查的主路径是：

1. 先保留论文原文中已提取到的 IF
2. 用 MJL 解析期刊元数据
3. 从 MJL profile 接口获取可见 IF
4. 如果 MJL 返回 `NO_MATCH`、`NOT_VISIBLE`、`ERROR` 等不可用结果，再走搜索抓取回退（Bing RSS + 出版商可信域）
5. 如果启用了 WOS Journals API，再尝试 WOS
6. 如果手工开启了实验性 `use_ai_model_if`，再优先进入 AI IF 模块

补充说明：

- `MJLImpactFactorFetcher` 内部还带一小组硬编码 fallback 期刊值。
- `impact_factor_status` 常见值包括 `OK`、`NO_MATCH`、`MULTI_MATCH`、`NO_ACCESS`、`NOT_VISIBLE`、`ERROR`。

## 8. 当前值已经有 IF，程序还会改吗？

可能会。

当 `web_search.correct_existing_impact_factor=true` 时，如果联网结果与当前值差异足够大，或当前值明显无效，程序会用联网结果覆盖现值。

## 9. 双语输出什么时候会询问？

只有在 LLM 可用时，CLI 才会在未显式传 `--bilingual` / `--no-bilingual` 的情况下询问一次是否启用双语输出。

如果当前走的是 `regex` 模式，双语后处理会被关闭。

## 10. 如何启用实验性 AI 影响因子获取？

把下面的键手工加入 `~/.paperinsight/config.yaml`：

```yaml
web_search:
  use_ai_model_if: true
  ai_model_if:
    enabled: true
    timeout: 60
    qianwen_api_key: ""
    kimi_api_key: ""
```

注意：这些键现在已经在 `DEFAULT_CONFIG` 里，默认都是关闭状态。

## 11. 为什么示例配置和实际默认值看起来不一样？

因为它们不是同一个来源：

- `paperinsight/utils/config.py::DEFAULT_CONFIG` 决定运行时默认值
- `config/config.example.yaml` 只是示例文件

比如当前运行时默认是 `mineru.mode=api`、`output.format=['excel']`，而示例文件给的是另一组推荐写法。

## 12. `paperinsight` 命令找不到怎么办？

可以直接用模块方式启动：

```bash
python3 -m paperinsight.cli analyze ./pdfs
```

## 13. 桌面版怎么启动？

开发模式：

```bash
cd desktop
npm install
npm run dev
```

Windows 打包流程可以看 `.github/workflows/build-desktop-windows.yml`。
