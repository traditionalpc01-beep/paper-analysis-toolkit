# 快速开始指南

这份指南按“检查环境 -> 配置能力 -> 运行分析 -> 查看结果”的顺序带你上手。

## 1. 安装

```bash
pip install -r requirements.txt
pip install -e .
```

如果你计划处理扫描版 PDF，建议额外安装本地 OCR：

```bash
pip install paddlepaddle paddleocr
```

如果要用 OpenAI 模型：

```bash
pip install openai
```

## 2. 启动前检查

```bash
paperinsight check
```

这一步会告诉你：

- Python 版本是否满足要求
- 核心依赖是否齐全
- 网络是否可用
- 本地 PaddleOCR 是否可用

如果想看更完整的诊断：

```bash
paperinsight doctor
```

## 3. 运行配置向导

```bash
paperinsight config
```

配置向导会按顺序询问你是否启用：

- PaddleX API
- LLM 提取
- Web 搜索补全影响因子

如果你暂时没有任何 API Key，也可以全部选择 `n`，之后先用 `auto` 或 `regex` 模式试跑。

## 4. 准备 PDF 文件

假设你的论文都放在 `./pdfs`：

```bash
mkdir -p pdfs
# 把 PDF 拷贝到 pdfs/
```

## 5. 开始分析

### 最常用的命令

```bash
paperinsight analyze ./pdfs
```

### 常见变体

```bash
# 递归分析子目录
paperinsight analyze ./pdfs --recursive

# 强制基础正则模式
paperinsight analyze ./pdfs --mode regex

# 强制 API 模式
paperinsight analyze ./pdfs --mode api

# 同时导出 JSON
paperinsight analyze ./pdfs --json

# 分析完成后重命名 PDF
paperinsight analyze ./pdfs --rename-pdfs

# 每篇只读前 5 页
paperinsight analyze ./pdfs --max-pages 5
```

## 6. 查看结果

默认输出目录是输入目录下的 `输出结果/`。

通常你会看到：

- `Paper_Analysis_Report.xlsx`
- `Paper_Analysis_Report.json`（如果加了 `--json`）
- `error_log.txt`（如果存在失败项）

推荐优先打开 Excel：

- 它会按“影响因子”降序排列
- 多器件参数会放在同一单元格中换行展示
- 可快速按期刊、EQE、寿命等维度筛选

## 7. 首次使用建议

如果你不知道该用哪个模式：

- 先运行 `paperinsight analyze ./pdfs`
- 如果结果不理想，再运行 `paperinsight config`
- 配完后再用 `paperinsight analyze ./pdfs --mode api`
