# 快速开始指南

本文按“安装 -> 检查 -> 配置 -> 分析 -> 查看结果”的顺序整理，只描述当前仓库中可以直接核查的行为。

## 1. 安装

### 基础安装

```bash
git clone https://github.com/traditionalpc01-beep/paper-analysis-toolkit.git
cd paper-analysis-toolkit
pip install -r requirements.txt
pip install -e .
```

### 按需补充依赖

```bash
# 如果要启用 OpenAI / DeepSeek / Longcat 的 LLM 提取
pip install openai

# 如果要启用本地 OCR 兜底
pip install paddlepaddle paddleocr

# 如果要把 MinerU 当作本地 CLI 使用
pip install mineru

# 实验性 AI 影响因子补全所需的 HTML 解析依赖已包含在基础安装中
```

## 2. 启动前检查

```bash
paperinsight check
```

当前 `check` 会检查：

- Python 版本是否至少为 3.9
- `typer`、`rich`、`PyMuPDF`、`openpyxl`、`pydantic` 是否可导入
- 基础网络是否可用
- 本地配置是否完整

如果要看更完整的诊断信息：

```bash
paperinsight doctor
```

## 3. 运行配置向导

```bash
paperinsight config
```

当前配置向导会：

- 先展示环境摘要
- 默认进入 Longcat 配置流程
- 再询问是否启用 MinerU，以及选择 `api` / `cli` 模式
- 最后保存到 `~/.paperinsight/config.yaml`

## 4. 准备 PDF

示例目录：

```bash
mkdir -p pdfs
# 把待分析的 PDF 放入 ./pdfs
```

## 5. 开始分析

### 最常用命令

```bash
paperinsight analyze ./pdfs
```

### 选择提取模板

PaperInsight 支持多种研究领域的提取模板：

```bash
# OLED/LED/QLED 器件（默认）
paperinsight analyze ./pdfs --template oled

# 太阳能电池（钙钛矿、有机太阳能电池等）
paperinsight analyze ./pdfs --template solar_cell

# 锂电池（锂离子电池、固态电池、锂硫电池等）
paperinsight analyze ./pdfs --template battery

# 传感器（电化学传感器、生物传感器、气体传感器等）
paperinsight analyze ./pdfs --template sensor
```

各模板提取的核心参数：

| 模板 | 核心器件参数 |
|------|-------------|
| `oled` | EQE、CIE、寿命、亮度、电流效率、功率效率 |
| `solar_cell` | PCE、Jsc、Voc、FF、稳定性、迟滞 |
| `battery` | 容量、循环稳定性、能量密度、功率密度、库伦效率 |
| `sensor` | 灵敏度、检测限、线性范围、选择性、响应时间 |

### 常见变体

```bash
# 递归扫描子目录
paperinsight analyze ./pdfs --recursive

# 强制正则兜底模式
paperinsight analyze ./pdfs --mode regex

# 强制 API 模式
paperinsight analyze ./pdfs --mode api

# 同时导出 JSON
paperinsight analyze ./pdfs --json

# 处理后重命名 PDF
paperinsight analyze ./pdfs --rename-pdfs

# 关闭缓存重跑
paperinsight analyze ./pdfs --no-cache

# LLM 启用时输出中英双语文本列
paperinsight analyze ./pdfs --bilingual

# 组合使用：太阳能电池论文 + 递归扫描 + JSON 导出
paperinsight analyze ./pdfs -t solar_cell -r --json
```

## 6. 查看结果

默认输出目录是输入目录下的 `输出结果/`。

当前实现会生成：

- `论文分析报告_<时间戳>.xlsx`
- `论文分析报告_<时间戳>.json`（仅在 `--json` 或配置中启用 `json` 时）
- `error_log.txt`（仅在有错误时）

Excel 报表默认按影响因子降序排序；多器件字段会在同一个单元格内用换行拼接。

## 7. 首次使用建议

- 没有任何 API Key 时，先跑 `paperinsight analyze ./pdfs --mode regex`。
- 准备开启 LLM / 联网补全时，先执行 `paperinsight config`。
- 如果你使用的是 `longcat`、`deepseek` 或 `openai`，请确认已经安装 `openai` Python SDK。

## 8. 桌面开发模式

仓库内带有桌面壳，开发模式命令如下：

```bash
cd desktop
npm install
npm run dev
```

Windows 安装包构建命令见 `desktop/package.json` 和 `.github/workflows/build-desktop-windows.yml`。
