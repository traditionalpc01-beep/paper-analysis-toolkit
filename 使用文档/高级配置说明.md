# 高级配置说明

本文优先以运行时代码 `paperinsight/utils/config.py::DEFAULT_CONFIG` 为准；`config/config.example.yaml` 只是仓库内的示例文件。两者存在差异时，应以运行时代码为准。

## 1. 配置文件位置与来源

- 运行时配置：`~/.paperinsight/config.yaml`
- 运行时默认值：`paperinsight/utils/config.py::DEFAULT_CONFIG`
- 仓库示例文件：`config/config.example.yaml`

建议先通过下面的命令生成本地配置，再手工微调：

```bash
paperinsight config
```

## 2. 当前运行时默认值摘要

| 配置组 | 当前默认值 | 说明 |
| --- | --- | --- |
| `mineru.enabled` | `true` | 默认启用 MinerU |
| `mineru.mode` | `api` | 运行时代码默认是 `api` |
| `llm.enabled` | `true` | 默认开启 LLM 流程 |
| `llm.provider` | `longcat` | 配置向导也默认走 Longcat |
| `output.format` | `['excel']` | 默认只导出 Excel |
| `output.rename_pdfs` | `false` | 默认不重命名 PDF |
| `output.bilingual_text` | `false` | 默认不启用双语后处理 |
| `web_search.enabled` | `true` | 默认允许联网补全 |
| `web_search.search_crawler.enabled` | `true` | 默认启用搜索抓取回退 |
| `web_search.web_of_science.enabled` | `false` | 默认不开启 WOS Journals API |
| `cache.enabled` | `true` | 默认启用缓存 |
| `cache.directory` | `.cache` | 默认缓存目录 |
| `cleaner.max_input_chars` | `24000` | 清洗后送入 LLM 的预算上限 |
| `pdf.text_ratio_threshold` | `0.1` | 基础 PDF 文本提取阈值 |

## 3. 示例配置与运行时默认值

当前 `config/config.example.yaml` 已按 `DEFAULT_CONFIG` 对齐整理，并保留了完整嵌套结构，方便直接对照字段含义。

如果你想知道程序“没改配置时”会怎么跑，请看 `DEFAULT_CONFIG`；如果你想看一份可直接参考的完整 YAML 结构，再看 `config/config.example.yaml`。

## 4. 关键配置组

### 4.0 `template_id`（提取模板）

当前支持四种研究领域模板：

```yaml
# 在配置文件中设置默认模板
template_id: oled  # oled / solar_cell / battery / sensor
```

各模板说明：

| 模板 ID | 研究领域 | 核心器件字段 |
|---------|----------|-------------|
| `oled` | OLED/LED/QLED 器件 | eqe, cie, lifetime, luminance, current_efficiency, power_efficiency |
| `solar_cell` | 钙钛矿/有机太阳能电池 | pce, jsc, voc, ff, stability, hysteresis |
| `battery` | 锂离子/固态/锂硫电池 | capacity, cycling_stability, energy_density, power_density, coulombic_efficiency |
| `sensor` | 电化学/生物/气体传感器 | sensitivity, detection_limit, linear_range, selectivity, response_time |

也可以通过命令行参数覆盖：

```bash
paperinsight analyze ./pdfs --template solar_cell
```

### 4.1 `mineru`

当前实现会读取这些键：

```yaml
mineru:
  enabled: true
  mode: api          # api 或 cli
  token: ""
  api_url: "https://mineru.net/api/v4"
  timeout: 600
  model_version: "vlm"
  output_format: "markdown"
  extract_tables: true
  method: "auto"
```

说明：

- `cli` 模式要求本机存在 `magic-pdf` 命令。
- `api` 模式要求 `token` 非空。
- `method` 会传给 `magic-pdf -m <method>` 或 MinerU API 请求。

### 4.2 `llm`

当前运行时默认结构：

```yaml
llm:
  enabled: true
  provider: longcat
  api_key: ""
  model: LongCat-Flash-Chat
  base_url: ""
  timeout: 120
  max_retries: 3
```

补充说明：

- `openai`、`deepseek`、`longcat` 这三条客户端路径当前都依赖 `openai` Python SDK。
- `wenxin` 额外使用 `llm.wenxin.client_id` 与 `llm.wenxin.client_secret`。
- `longcat.backfill_model` 与 `longcat.enable_lite_backfill` 用于轻量补全标题/期刊/年份等缺失字段。

### 4.3 `multi_turn`（多轮对话验证）

多轮对话验证可以在首次提取后检查关键字段是否缺失，如有缺失则自动构造补充提问：

```yaml
multi_turn:
  enabled: false       # 是否启用多轮验证
  enable_follow_up: true  # 是否允许补充提问
  max_follow_ups: 1    # 最大补充提问轮数
```

说明：

- 启用后，系统会检查各模板的关键字段（如 OLED 的 EQE、结构、CIE）
- 如果关键字段缺失，会自动构造补充提问让 LLM 重新提取
- 适用于提取准确率要求较高的场景
- 会增加 LLM 调用次数和耗时

各模板关键字段定义：

| 模板 | 关键字段 |
|------|----------|
| `oled` | eqe, structure, cie |
| `solar_cell` | pce, jsc, voc, ff |
| `battery` | capacity, cycling_stability |
| `sensor` | sensitivity, detection_limit, target_analyte |

### 4.4 `web_search`

当前运行时默认结构：

```yaml
web_search:
  enabled: true
  timeout: 30
  resolve_journal_metadata: true
  fetch_official_impact_factor: true
  correct_existing_impact_factor: true
  search_crawler:
    enabled: true
    market: en-US
  web_of_science:
    enabled: false
    api_key: ""
    journals_api_url: "https://api.clarivate.com/apis/wos-journals/v1"
```

说明：

- `resolve_journal_metadata` 控制是否先走 MJL 期刊解析。
- `fetch_official_impact_factor` 控制是否继续查询 MJL/WOS/搜索回退。
- `correct_existing_impact_factor` 为真时，联网结果可能覆盖原文提取值。
- `search_crawler` 当前实现使用 Bing RSS 搜索，再只信任若干出版商域名页面。
- `web_of_science` 只有在 `enabled=true` 且 `api_key` 非空时才会初始化。

### 4.5 `output`

当前运行时默认结构：

```yaml
output:
  format: [excel]
  sort_by_if: true
  generate_error_log: true
  rename_pdfs: false
  rename_template: "[{year}_{impact_factor}_{journal}]_{title}.pdf"
  include_source: true
  bilingual_text: false
```

说明：

- `format` 只控制最终导出，不改变中间处理逻辑。
- `bilingual_text` 只有在 LLM 可用时才真正生效。
- 报表默认按影响因子降序排序。

### 4.6 `cleaner`

当前运行时默认结构：

```yaml
cleaner:
  enabled: true
  block_window: 1
  max_input_chars: 24000
  max_blocks: 80
  min_block_score: 3.0
  keep_table_context: true
```

说明：

- 当前清洗器不是简单的“章节白名单截断”，而是 Markdown 块切分、打分、邻域保留、表格锚点化。
- `remove_sections` / `keep_sections` 仍可作为额外的噪声识别与保留提示。

## 5. 实验性配置：`use_ai_model_if`

`paperinsight/core/pipeline.py` 会读取下面这些实验性键：

```yaml
web_search:
  use_ai_model_if: true
  ai_model_if:
    enabled: true
    timeout: 60
    qianwen_api_key: ""
    kimi_api_key: ""
```

注意：

- 这些键现在已包含在 `DEFAULT_CONFIG` 中，默认值都是关闭态。
- `config/config.example.yaml` 与运行时代码的字段结构保持一致。
- 当前 AI IF 模块仍是实验性路径，详见 [AI模型影响因子获取说明](./AI模型影响因子获取说明.md)。

## 6. 配置保存与安全

当前实现会：

- 把敏感字段（如 `token`、`api_key`、`client_id`、`client_secret`）写入前做本地加密/混淆。
- 使用 `~/.paperinsight/.key` 保存本地密钥。
- 尝试把 `config.yaml` 和 `.key` 的权限都设置为 `0600`。

这套方案是“本机文件级保护”，不是硬件安全模块，也不是操作系统凭据保险箱。
