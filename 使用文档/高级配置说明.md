# 高级配置说明

这份文档说明 `~/.paperinsight/config.yaml` 的主要配置项，以及它们对运行行为的影响。

## 配置文件位置

默认配置文件路径：

```text
~/.paperinsight/config.yaml
```

推荐优先通过命令生成：

```bash
paperinsight config
```

## 配置结构总览

```yaml
paddlex:
  enabled: false
  token: ""
  model: "PaddleOCR-VL-1.5"
  use_doc_orientation: false
  use_doc_unwarping: false
  use_layout_detection: true
  use_chart_recognition: false
  timeout: 300
  poll_interval: 5

llm:
  enabled: false
  provider: "openai"
  api_key: ""
  model: "gpt-4o"
  base_url: ""
  timeout: 120

web_search:
  enabled: true
  timeout: 30

cache:
  enabled: true
  directory: ".cache"

output:
  format: ["excel"]
  sort_by_if: true
  generate_error_log: true
  rename_pdfs: false
  rename_template: "[{year}_{impact_factor}_{journal}]_{title}.pdf"

pdf:
  max_pages: 0
  text_ratio_threshold: 0.1
```

## OCR 配置

### `paddlex`

适合：

- 使用百度 AI Studio 提供的 PaddleX API
- 需要更强的版面解析或结构化能力

关键字段：

- `enabled`：是否启用
- `token`：AI Studio Token
- `model`：模型名，如 `PaddleOCR-VL-1.5`
- `poll_interval`：轮询间隔
- `timeout`：接口超时时间

说明：

- 如果未启用 PaddleX，程序会优先尝试原生文本提取，并在必要时依赖本地 OCR 或直接回落。

## LLM 配置

### `llm`

关键字段：

- `enabled`：是否启用
- `provider`：`openai` 或 `deepseek`
- `api_key`：对应服务的 API Key
- `model`：模型名
- `base_url`：可选，用于代理或兼容端点
- `timeout`：请求超时秒数

建议：

- 先用默认模型试跑
- 稳定后再根据成本和效果调整模型

## Web 搜索配置

### `web_search`

- `enabled`：是否允许联网补全影响因子
- `timeout`：超时秒数

如果关闭：

- 程序只会使用论文原文或本地提取结果中的 IF
- 不再尝试联网补全

## 缓存配置

### `cache`

- `enabled`：是否启用缓存
- `directory`：缓存目录，默认 `.cache`

建议默认开启，因为可以复用：

- OCR 文本缓存
- 结构化数据缓存

## 输出配置

### `output`

- `sort_by_if`：是否按影响因子降序排序
- `rename_pdfs`：是否默认重命名处理后的 PDF
- `rename_template`：重命名模板

支持的模板变量：

- `{year}`
- `{impact_factor}`
- `{if}`
- `{journal}`
- `{title}`

例如：

```yaml
output:
  rename_pdfs: true
  rename_template: "[{year}_{impact_factor}_{journal}]_{title}.pdf"
```

## PDF 处理配置

### `pdf`

- `max_pages`：默认每篇最多读取多少页，`0` 表示不限制
- `text_ratio_threshold`：原生文本提取时用于判断文本层密度的阈值

经验建议：

- 快速抽样时可把 `max_pages` 设为 `5`
- 如果扫描件很多，建议至少配置一种 OCR 能力

## 推荐工作流

### 方案一：先试跑再增强

```bash
paperinsight analyze ./pdfs
paperinsight config
paperinsight analyze ./pdfs --mode api
```

### 方案二：大批量正式处理

```bash
paperinsight check
paperinsight config
paperinsight analyze ./pdfs --recursive --json --rename-pdfs
```
